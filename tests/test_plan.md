# Checklist Breve de Pruebas y Entrega

## 1) Checklist de pruebas mínimas
- [ ] Ejecuté el programa al menos una vez sin errores críticos.
- [ ] Probé al menos dos entradas válidas.
- [ ] Probé al menos una entrada inválida.
- [ ] Confirmé que el resultado coincide con lo esperado.

## 2) Checklist de documentación
- [ ] Completé `docs/propuesta.md`.
- [ ] Completé `docs/caso_de_uso.md`.
- [ ] Completé `docs/plan_de_pruebas.md`.
- [ ] Revisé consistencia entre documentación y prototipo.

## 3) Checklist de ejecución
- [ ] Verifiqué que `scripts/run.sh` funciona o lo adapté a mi lenguaje.
- [ ] El archivo principal en `src/` existe y está actualizado.
- [ ] No dependo de servicios externos para ejecutar lo básico.

## 4) Checklist de entrega final
- [ ] El proyecto se mantiene pequeño y viable.
- [ ] No usé frameworks, nube, Docker ni bases de datos.
- [ ] El repositorio está ordenado y sin archivos basura.
- [ ] La propuesta está lista para revisión en GitHub Classroom.
