# Actividad de GitHub Classroom: Propuesta de Práctica Temática (Proyecto Pequeño)

## Descripción general
En esta actividad vas a diseñar una propuesta de práctica temática enfocada en resolver una necesidad concreta con un proyecto pequeño de consola. El enfoque principal es la documentación técnica, la planeación del alcance y la justificación de decisiones antes de programar.

## Objetivo de aprendizaje
Al finalizar esta actividad, el estudiante será capaz de:
- Definir un problema técnico acotado y viable.
- Justificar la elección de lenguaje según el caso de uso.
- Diseñar una estructura clara de repositorio para GitHub Classroom.
- Documentar alcance, pruebas y criterios de éxito.
- Construir, de forma opcional, un prototipo mínimo funcional.

## Lenguajes permitidos
- ARM64 Assembly
- C
- Python
- Bash

## Reglas para mantener el proyecto pequeño
- Enfócate en una sola necesidad principal.
- Máximo 1 archivo principal en `src/` durante la primera entrega.
- Evita interfaces gráficas, frameworks y arquitecturas complejas.
- No uses bases de datos, nube, contenedores ni APIs externas.
- Diseña funcionalidades mínimas pero bien documentadas.
- Si eliges ARM64 Assembly, limita el alcance a programas muy pequeños de consola.

## Entregables esperados
1. `docs/propuesta.md` completo.
2. `docs/caso_de_uso.md` completo.
3. `docs/plan_de_pruebas.md` completo.
4. `tests/test_plan.md` con checklist final.
5. Estructura del repositorio ordenada y consistente.
6. (Opcional) Prototipo mínimo en `src/main.<ext>`.

## Instrucciones para el estudiante
1. Lee este `README.md` completo.
2. Elige un tema pequeño (ejemplos abajo).
3. Define el lenguaje principal y justifícalo en `docs/propuesta.md`.
4. Describe el caso de uso real en `docs/caso_de_uso.md`.
5. Documenta cómo vas a probar tu solución en `docs/plan_de_pruebas.md`.
6. Verifica que tu estructura cumpla con `docs/estructura_repositorio.md`.
7. Si te alcanza el tiempo, crea un prototipo mínimo funcional.

## Ejemplos de posibles temas
- Mini Toolkit en ARM64
- Asistente de Estudio en Terminal
- Reporteador de Información del Sistema
- Organizador de Archivos
- Juego de Aprendizaje en Línea de Comandos

## Criterios generales de evaluación
- Claridad del problema y viabilidad del alcance.
- Coherencia entre lenguaje elegido, caso de uso y solución propuesta.
- Calidad de la documentación técnica.
- Orden de repositorio y buenas prácticas básicas.
- Calidad de pruebas documentadas.
- (Opcional) Funcionamiento del prototipo mínimo.

## Nota importante
Primero se documenta y justifica la idea. Después, de manera opcional, se implementa un prototipo pequeño. En esta actividad vale más una propuesta clara y viable que mucho código sin planeación.
